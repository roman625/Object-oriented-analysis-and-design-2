# Лабораторная работа — Паттерн «Value Object» (Объект-значение)

## Предметная область: Система управления космической торговлей

### Описание предметной области

Приложение представляет собой симулятор космического торговца. Игрок перемещается между планетами, покупает и продает товары, а также следит за запасом топлива. Важной частью системы является обработка точных финансовых расчетов и географических координат в космическом пространстве.

| Критерий | Описание | Примеры сущностей |
| --- | --- | --- |
| **Координаты** | Позиция планеты в 2D-пространстве | `Position(x, y)` |
| **Финансы** | Денежные средства игрока (кредиты) | `Currency(amount)` |
| **Логика перемещения** | Расчет дистанции и затрат топлива | Расстояние между `Position` |

**Реализованные сущности-значения:**

| Сущность | Роль в паттерне | Функционал | Свойство паттерна |
| --- | --- | --- | --- |
| **Position** | Объект-значение | Расчет евклидова расстояния | **Immutability** (неизменяемость) |
| **Currency** | Объект-значение | Безопасные расчеты с `BigDecimal` | **Self-validation** (инкапсуляция логики) |

---

### Описание проблемы (без паттерна)

В реализации без использования объектов-значений данные «рассыпаны» по примитивным полям. Это приводит к раздуванию логики в классах-контроллерах и дублированию кода:

```java
// Плохой пример: логика расчета размыта по приложению
public boolean flyTo(Planet target) {
    // Формула расчета дистанции прописана прямо в методе перемещения
    double dist = Math.sqrt(Math.pow(currentPlanet.getX() - target.getX(), 2) + 
                            Math.pow(currentPlanet.getY() - target.getY(), 2));
    // ... логика топлива
}

// В другом месте (например, в отрисовке GUI) эту формулу приходится писать заново
double d = Math.sqrt(Math.pow(shipX - pX, 2) + Math.pow(shipY - pY, 2));

```

**Проблемы такого подхода:**

| Проблема | Последствия |
| --- | --- |
| **Нарушение DRY** | Математические формулы дублируются в разных частях программы. |
| **Примитивная одержимость** | Использование `double` или `int` вместо семантических типов делает код менее понятным. |
| **Риск ошибок** | Легко перепутать `x` и `y` или забыть о правилах округления валюты. |

---

### Решение: применение паттерна «Value Object»

#### 1. Использование Java Records для неизменяемости

Координаты и валюта представлены как `record`. Это гарантирует, что объект нельзя изменить после создания — его можно только заменить новым.

```java
public record Position(int x, int y) {
    public double distanceTo(Position other) {
        return Math.sqrt(Math.pow(x - other.x, 2) + Math.pow(y - other.y, 2));
    }
}

```

#### 2. Инкапсуляция бизнес-логики (Currency)

Объект-значение берет на себя ответственность за математические операции, скрывая сложность работы с `BigDecimal`.

```java
public record Currency(BigDecimal amount) {
    public Currency add(BigDecimal other) { 
        return new Currency(amount.add(other)); 
    }
    public boolean isGreaterThanOrEqual(BigDecimal other) { 
        return amount.compareTo(other) >= 0; 
    }
    @Override 
    public String toString() { return String.format("%.2f cr", amount); }
}

```

#### 3. Упрощение основного кода (Ship)

Благодаря объектам-значениям, класс `Ship` (сущность) занимается только высокоуровневой логикой:

```java
public boolean flyTo(Planet target) {
    // Корабль не знает, как считать дистанцию, он просит об этом Позицию
    double dist = currentPlanet.pos.distanceTo(target.pos); 
    double cost = dist * FUEL_RATE;
    if (fuel >= cost) { /* ... */ }
}

```

### Вывод: преимущества применения Value Object

| Преимущество | Как реализовано в коде |
| --- | --- |
| **Чистота кода** | Основные классы избавились от громоздких математических формул. |
| **Типобезопасность** | Мы не можем случайно передать координаты туда, где ожидается цена, так как это разные типы. |
| **Удобство тестирования** | Логику `distanceTo` или `add` в `Currency` можно тестировать отдельно от всего приложения. |
| **Семантика** | Метод `wallet.isGreaterThanOrEqual(price)` читается как живой язык по сравнению с `compareTo`. |
