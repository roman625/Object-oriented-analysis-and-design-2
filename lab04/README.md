## 1. SpaceTraderApp (Главный контроллер / Оркестратор)
Этот класс управляет жизненным циклом приложения и связывает модель с интерфейсом.

### Поля (Attributes):
* **`- <u>WIN_GOAL: BigDecimal</u>`**: 
    * **Модификатор `-`**: `private`. Поле инкапсулировано, доступ к нему извне закрыт для предотвращения случайного изменения логики победы.
    * **Подчеркивание**: `static`. Переменная принадлежит классу, а не конкретному объекту. Существует в единственном экземпляре.
    * **Тип `BigDecimal`**: Используется для хранения финансовых целей с произвольной точностью, чтобы избежать ошибок округления, которые возникают у типов `float` или `double`.
    * **Назначение**: Константа, определяющая порог капитала для завершения игры.
* **`- priceModel: DefaultTableModel`**:
    * **Модификатор `-`**: `private`. Объект таблицы скрыт внутри приложения.
    * **Тип `DefaultTableModel`**: Стандартный класс Swing для управления данными в таблице.
    * **Назначение**: Хранит актуальные цены планет для отображения пользователю.

### Методы (Operations):
* **`+ main(args: String[])`**: Публичный статический метод. Точка входа в программу для виртуальной машины Java (JVM).
* **`+ gui()`**: Публичный метод инициализации графического интерфейса (настройка окон, кнопок, слоев).
* **`+ refresh()`**: Публичный метод для обновления состояния UI. Вызывается всякий раз, когда меняются данные в модели (баланс, топливо).

## 2. Ship (Модель / Singleton)
Центральный объект логики, реализующий паттерн «Одиночка».

### Поля (Attributes):
* **`- <u>instance: Ship</u>`**: 
    * **Подчеркивание**: `static`. Хранит ссылку на единственный созданный объект корабля.
    * **Назначение**: Реализация Singleton.
* **`- fuel: Real`**: Приватное числовое поле с плавающей запятой. Хранит текущий запас энергии.
* **`- cargoCount: Integer`**: Целочисленное приватное поле. Счётчик единиц товара в трюме.
* **`- fuel_rate: Real`**: Статический коэффициент расхода топлива на единицу расстояния.
* **`- max_fuel: Integer`**: Константа, ограничивающая максимальный объём бака.

### Методы (Operations):
* **`- Ship()`**: Приватный конструктор. Запрещает создание объектов через `new` за пределами класса, гарантируя единственность экземпляра.
* **`+ getInstance(): Ship`**: Публичный статический метод (фабричный метод). Возвращает ссылку на единственный объект `Ship`.
* **`+ flyTo(target: Planet): Boolean`**: Бизнес-метод. Принимает объект планеты-цели, рассчитывает дистанцию и остаток топлива. Возвращает `true`, если перемещение выполнено.
* **Геттеры и Сеттеры (`+ getFuel()`, `+ setShip()` и др.)**: Публичные методы доступа к скрытым полям. Позволяют безопасно читать и изменять состояние корабля (например, обновить координаты при посадке).

## 3. Currency, Planet, Position (Value Objects)
Специализированные структуры данных для хранения информации.

### Currency (Деньги):
* **`+ amount: BigDecimal`**: Публичное поле суммы. Использование `BigDecimal` здесь критично: финансовые операции требуют абсолютной точности (копейка к копейке).
* **`+ add() / subtract()`**: Методы арифметики. Обратите внимание, они возвращают новый объект `Currency`, реализуя принцип **Immutability** (неизменяемости).

### Planet (Планета):
* **`+ pos: Position`**: Поле типа `Position`. Связывает планету с её местоположением.
* **`+ color: Integer`**: Хранит цветовой код (например, HEX или ARGB) для визуализации на карте.

### Position (Координаты):
* **`+ x, y: Integer`**: Целочисленные координаты в двумерном пространстве.
* **`+ distanceTo(other: Position): Real`**: Метод вычисления расстояния между точками по формуле Евклида: $d = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$.

## 4. Связи между классами (Relationships)

1.  **Композиция (Черный ромб ◆)**:
    * `Ship` ◆— `Currency`: Корабль владеет кошельком. Если уничтожить корабль, кошелек тоже перестанет существовать. Это жесткая связь «целое-часть».
    * `Planet` ◆— `Position`: Планета не существует без координат.
    * `SpaceTraderApp` ◆— `MapPanel` / `Planet`: Приложение является владельцем панели карты и списка планет.
2.  **Зависимость (Пунктирная стрелка с `<<использует>>`)**:
    * `SpaceTraderApp` --> `Ship`: Приложение вызывает методы корабля (например, `Ship.getInstance()`), но не «владеет» им в классическом смысле, так как корабль — синглтон.
3.  **Ассоциация / Наблюдение (Пунктир с `<<наблюдает>>`)**:
    * `MapPanel` --> `Ship`: Панель должна знать состояние корабля, чтобы отрисовать его текущую позицию.
4.  **Ассоциация / Отрисовка (Пунктир с `<<рисует>>`)**:
    * `MapPanel` --> `Planet`: Панель берет данные о цветах и координатах планет для вывода графики на экран.
5.  **Навигация (Сплошная линия со стрелкой)**:
    * `Ship` --> `Planet`: Корабль хранит ссылку на текущую планету, на которой он приземлился.
6.  **Циклическая связь (Петля на Ship)**:
    * Указывает на то, что внутри класса `Ship` есть поле, тип которого — `Ship` (это и есть статический `instance`).
